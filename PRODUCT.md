# Product

## Register

product

## Users

墨鉴面向两类用户。

普通用户以聊天方式咨询事业、感情、财务、性格等问题。他们通常不熟悉八字术语，也不希望先面对复杂排盘工作台。他们需要的是清楚、克制、能落到现实选择上的分析。

运营和内部专家使用后台 trace 来复核每一轮回答。他们需要通过分析 ID 找到原始对话、出生信息抽取、排盘、诊断、仲裁、LLM 输入输出，以及最终回复的生成依据。

## Product Purpose

墨鉴是建立在 `bazibase` 确定性规则引擎上的聊天式八字命理咨询产品。

前台的目标是让普通用户像聊天一样提问，并获得简洁、专业、不过度断言的命理分析。排盘、规则诊断、仲裁和证据链隐藏在产品背后，不作为普通用户的主要界面。

后台的目标是让运营和专家能够用分析 ID 精准追踪每一轮回答，判断问题来自用户信息、规则诊断、仲裁、模型生成，还是前台表达。

成功标准是：用户能自然发问，回答有专业感但不压迫；内部能追溯每个结论的来源；核心命理判断稳定遵循既定流派。

## Brand Personality

沉静、专业、可信、克制。

语气应像一位审慎的命理顾问：不神秘化、不恐吓、不夸张承诺，也不把后台规则链直接倾倒给普通用户。专业感来自判断边界、术语控制和可追溯性，而不是玄学包装。

## Anti-references

不要像常见的“玄机”“天机”类网站：重红金装饰、强神秘暗示、大量玄学符号、夸张标题、命运恐吓、诱导付费、信息堆叠、难以验证的断语。

不要把排盘工作台作为对外默认体验。普通用户不应先看到四柱表、规则 JSON、仲裁 schema、模型 prompt 或完整命理术语链。

不要做营销页式夸张 hero，也不要把产品包装成全知的算命大师。回答必须保留不确定性和现实建议边界。

## Design Principles

1. 聊天优先：用户的第一屏应该是可直接提问的咨询体验，而不是表单或排盘报告。
2. 专业藏在背后：前台少量使用专业语言，后台完整保留规则、trace 和证据链。
3. 流派约束藏在后台：前台不展示具体流派名、古籍书名和规则来源，后台保留完整判断依据。
4. 谨慎表达：不做绝对化预测，不做健康诊断，不给具体投资指令，不用恐吓式语言。
5. 可追溯：每一轮 assistant 回复都有分析 ID，运营可以定位原始对话、工具调用、诊断结果、仲裁和模型输出。

## Accessibility & Inclusion

默认按 WCAG AA 目标设计。

正文和关键操作需要保持清晰对比度。移动端应能完成完整聊天流程。动效只用于状态反馈，避免长时间、强干扰或无法关闭的装饰动画，并尊重用户的 reduced motion 偏好。

命理表达应避免恐吓、羞辱、宿命化和对重大人生决定的命令式建议。用户应能理解：产品提供的是基于既定规则体系的分析参考，不替代个人判断、法律、医疗或财务建议。
